Games we built together — pick a tab to play

Snake

Arrow keys or WASD to move. Eat the red squares, don't run into yourself or the walls. Built with Python + Pygame, compiled to run right in the browser.

Want to change it? Edit games/snake/main.py — try changing STARTING_SPEED or the colors at the top of the file.

Rat Cat Toe

Classic two-player, pass-and-play. Built with plain HTML/JavaScript — no compiling needed, so it's a great first game to customize.

Want to change it? Edit games/tictactoe/index.html — try changing the colors, the board size, or add a score counter.

Ratty Boi Racing

3D multiplayer rat-racing — create a room, share the code, and race friends on your home network. Built with Three.js + Socket.IO.

⚠️ This game needs its server running on this computer first. Start it with: node server/server.js from the Claude Code Work project folder, then come back to this tab. Unlike Snake and Rat Cat Toe, this tab won't work if the site is published to GitHub Pages without that server also running somewhere reachable — see the "Add a Game" tab for why.

Want to change it? Edit the files in Claude Code Work/public/ and Claude Code Work/server/server.js.

Paws & Claws Defense

3-player tower defense — Cats vs Rats vs Dogs. Build towers to defend your base, earn gold from kills, and spend it to send pests at your rivals' lanes. Built with Node.js + Socket.IO.

Hosted on Render, so it works for anyone — not just on your home network. It opens in its own tab rather than being embedded here, since it's a full multiplayer match screen, not a mini preview. Free-tier hosting means the first load after a while can take 30-60 seconds to wake up.

Adding a New Game

This site has two ways to build a game, depending on how ambitious you're feeling:

Option A — Simple: plain HTML/JavaScript

Good for board games, quizzes, memory match, word games. No build step required.

  1. Make a new folder in games/, e.g. games/memory/.
  2. Put a self-contained index.html in it (see games/tictactoe/index.html as a template).
  3. Add a new tab button and panel in index.html pointing at it (copy the Puzzle tab as a template).
  4. No build step needed — it just works.

Option B — Python + Pygame

Good for anything with movement, collisions, or real-time action — like Snake.

  1. Make a new folder in games/, e.g. games/pong/, with a main.py.
  2. Write it as a normal Pygame game, but use the async loop pattern from games/snake/main.py (required so it can run in a browser tab).
  3. Add a build step for it in .github/workflows/deploy.yml (copy the Snake build step).
  4. Add a new tab in index.html pointing at games/pong/index.html.

Option C — Networked/multiplayer game (needs its own server)

Good for real-time multiplayer, like the Racing tab. This is the odd one out: GitHub Pages only serves static files, so it can't run a live server for you. A game like this needs a real server (Node.js, Socket.IO, etc.) running somewhere — either on your own computer while you play, or on a separate always-on host if you want it reachable outside your home network.

  1. Build the game as its own standalone project with a server (see the Racing game's server/server.js as an example).
  2. Point a tab's iframe at wherever that server is running, e.g. http://<your-computer's-LAN-IP>:3000.
  3. Start the server before using that tab — this hub doesn't start it for you.

See the top-level README.md for full local setup and deployment instructions.
